## 1. 문제 정의

과제 원문은 "마케팅 성과 인사이트 리포트 생성"이지만, 실제로 풀어야 할 문제를 다음과 같이 재정의했다.

> 숫자를 예쁘게 요약하는 게 아니라, **리포트를 받은 사람이 그 자리에서 '이번 주 예산을 어디로 옮기고, 무엇을 손대지 말아야 하는지'를 결정하게 만드는 것** — 리포트와 의사결정 사이의 간극을 없애는 것.

주어진 데이터를 naive하게 "총매출·ROI 표"로만 요약하면, 이 데이터셋 자체가 세 번의 잘못된 결정을 유도한다.

| # | 함정 | 이 데이터에서 실제로 벌어지는 일 | 대응 설계 |
|---|------|-------------------------------|-----------|
| 1 | **액션 부재** | 총매출 4.4억·MER 862%를 봐도 "이번 주에 뭘 하라는 건가"가 없다 → 리포트를 받고 다시 회의를 열어야 함 | '이번 주 업무 우선순위' 표로 조치·담당·시점 확정 |
| 2 | **ROI 절대순위 오판** | 이메일이 ROI 1025.98%로 절대 1위 → "증액"이 자연스럽지만, 시장 벤치마크(ROAS) 대비로는 **개선여지** 등급 | 채널 평가를 내부 순위가 아닌 **시장 벤치마크 대비**로 판정 |
| 3 | **손실과 데이터 왜곡의 혼동** | 이메일 매출이상치 2,884만원을 '손실'로 읽으면 엉뚱한 예산 삭감 → 실제로는 원본 재확인이 필요한 **데이터 왜곡**(예산 손실 아님) | 이슈를 조치처(레버) 기준으로 예산 vs 데이터·트래킹으로 분리 |

**성공 기준.** 그래서 '잘함'의 기준을 "수치가 많다"가 아니라 **"받은 사람이 추가 회의 없이 이번 주 액션을 결정할 수 있다"**로 두고, 모든 섹션 설계를 이 기준에 역산해 맞췄다.

## 2. 데이터

- **출처·형식**: `data/marketing_performance.csv` — 채널별·일별 마케팅 성과 (노출·클릭·전환·광고비·매출)
- **규모**: 원본 280행, 채널 5개(네이버광고·메타광고·카카오광고·이메일 + 오가닉)
- **기간**: 2026-04-01 ~ 2026-05-26 (8주, W1–W8)
- **운영 규칙**: 매번 가장 최근 8주만 입력받는 것으로 고정 — 8주 초과 CSV는 자동으로 최근 8주만 트림(롤링 윈도우)

### 데이터 품질 처리 (EDA → 정제)

| 이슈 | 처리 | 왜 이렇게 했나 |
|------|------|--------------|
| 완전 중복 1행 | 제거 (280→279행) | 동일 레코드 중복은 집계를 부풀림 |
| 결측 3건 | **0으로 채우지 않고 NaN 유지**, 집계에서 자동 제외 | 0으로 채우면 평균·ROI가 하향 왜곡됨 |
| 매출 이상치 1건 | 해당 행의 revenue만 제외(정상 지출·전환은 보존) | AOV 기준 modified z-score > 3.5로 자동 탐지 — 한 컬럼 오류로 전체 행을 버리지 않기 위함 |
| 오가닉(광고비 0) | ROI 순위에서 제외 | 광고비 0은 ROI 측정 불가(0으로 나눔) |

## 3. 분석 방법론

### 파이프라인 아키텍처

```
data/marketing_performance.csv
        │
        ▼
┌───────────────────────────────────────────────────────┐
│ reallocate.run_pipeline()  — 정본 계산 1벌               │
│   ├─ calculate.py     합계·ROI·CTR·CVR·CPA·변화율·ROAS   │
│   │                   최근 8주 롤링 윈도우 트림           │
│   ├─ detect_issues.py 급등/급락/이상치/결측/기회 자동 탐지 │
│   └─ 예산 재원(회수 가능액) 산출 → 우선순위 재배분안 매칭   │
└───────────────────────────────────────────────────────┘
        │  (동일한 결과 dict를 두 렌더러가 공유)
        ├───────────────────────────┬─────────────────────
        ▼                           ▼
 generate_report.py           generate_html.py
 (markdown 렌더)                (정적 html 렌더)
        │                           │
        ▼                           ▼
 insight_report.md / .html    exec_report.md / .html
 (마케팅팀 상세본)              (경영진 요약본)
```

### 왜 이 방법을 선택했나

- **이상치 탐지 — modified z-score (median·MAD 기반, threshold 3.5).** 평균·표준편차 기반 z-score는 이상치 자신이 평균을 끌어당겨 둔감해진다. 중앙값·MAD 기반은 소수 데이터·극단값에 강건해 마케팅 일별 데이터처럼 튀는 값이 섞인 경우에 적합하다.
- **급등/급락 탐지 — LOO(leave-one-out) 중앙값 대비.** "이번 주가 유독 큰가"를 판정할 때 이번 주 자신을 포함해 비교하면 기준선이 오염된다. 판정 대상 주차를 제외한 나머지 주차의 중앙값을 기준선으로 삼고, 그 대비 배수가 적응형 임계값을 넘는지로 판정한다.
- **판정(강건)과 감시(민감)를 다른 축으로 분리.** 이상치·이슈 탐지는 강건한 *중앙값 대비*, 모멘텀 모니터링은 민감한 *전주 대비(WoW)*. WoW만으로 이상치를 판정하면 전주가 우연히 튈 때 왜곡되므로, WoW는 감시 전용으로만 쓴다.
- **전체 ROI를 3종으로 분리** — MER(오가닉 포함, 경영진용)·유료 순효율(오가닉 제외, 마케팅팀용)·오가닉 기여. 하나의 "전체 ROI"는 오가닉 포함 여부에 따라 862% vs 658%로 갈려 반드시 오해를 낳는다.
- **채널 진단에 CPA 추가** — ROI·ROAS는 "몇 배"라는 비율이라 "전환 하나에 얼마 썼나"라는 입찰가 조정의 원(₩) 단위 지표가 안 드러난다. CPA로 "그래서 얼마를 조정하나"에 답한다.

(지표 선택의 상세 근거는 [docs/REPORT_DESIGN.md](docs/REPORT_DESIGN.md) §2에 정리)

## 4. 핵심 분석 결과 (Key Findings)

**전체 성과 (8주 합계)**

| 지표 | 값 |
|------|-----|
| 총 매출 (오가닉 포함) | 439,553,059원 |
| └ 오가닉(비유료) 매출 | 104,238,814원 (23.71%) |
| 총 유료 광고비 | 50,992,739원 |
| 전체 마케팅 효율 MER | 861.99% |
| 유료 광고 순효율 | 657.57% |
| 총 전환 | 22,858건 |

**발견 1 — 회수 가능한 예산 손실이 존재한다.** 메타광고에서 W5 예산 과집행으로 **882만원 기회손실**이 발생했다. 이 초과 지출분 중 회수 가능액 **319만원**을, 시장 벤치마크 대비 우수한 네이버광고(ROAS 6.59 vs 메타 2.59)로 재배분하면 선형확장 가정 시 순증 매출 상한 **1,277만원**이 기대된다. → 재배분 기획안으로 연결.

**발견 2 — ROI 절대 1위 채널이 증액 대상이 아니다.** 이메일은 ROI 1025.98%로 절대 1위지만, 자기 채널 시장 벤치마크 대비로는 **개선여지** 등급이다. 절대 순위만 보고 증액하면 오판 — 증액 보류하고 오퍼·객단가·세그먼트 고도화로 효율 개선을 우선한다.

**발견 3 — '손실'이 아니라 '데이터 왜곡'인 이슈 2건.** 이메일 매출이상치 2,884만원, 네이버광고 revenue결측 139만원은 예산 손실이 아니라 원본 재확인이 필요한 데이터 왜곡이다. 예산 조치가 아니라 트래킹팀 확인 사안으로 분리해, 엉뚱한 예산 삭감을 방지한다.

|--------|------|---------|

**핵심 실행 산출물 — '이번 주 업무 우선순위' 표.** '핵심 이슈'의 근거를 순위·담당·시점으로 취합해, 이 표 하나로 팀 업무 분배가 끝나도록 설계했다.

| 순위 | 할 일 | 근거 | 담당 | 시점 |
|------|-------|------|------|------|
| 1 | 메타광고 과집행 차단 후 회수분을 네이버광고에 재배분 | 초과지출 3,194,281원, 순증 상한 12,772,685원 | 마케팅 리드 | 즉시 |
| 2 | 이메일 매출이상치 원본 재확인·보정 | 왜곡·추정 28,838,361원 (예산 손실 아님) | 트래킹팀 | 이번 주 |
| 3 | 네이버광고 revenue결측 원본 재확인·보정 | 왜곡·추정 1,385,548원 (예산 손실 아님) | 트래킹팀 | 이번 주 |

**예산 재배분 기획안**은 문제 정의 → 재원(회수 가능 손실) → 재배분안 → 우선순위 판단 기준까지 담아, 마케터가 추가 회의 없이 착수하게 만들었다. 우선순위는 새 스코어를 만들지 않고 채널 ROI·이슈 임팩트·벤치마크 등급 등 이미 계산된 값만 사용한다.

## 6. 재현성 & 검증

새 성과 CSV를 넣으면 정제 → 계산 → 이슈 탐지 → 리포트 생성이 동일 품질로 자동 재현된다. 주차·채널·기간이 달라도 하드코딩 없이 적응한다.

이를 **"된다"가 아니라 "자동 검증됨"으로 증명**하기 위해, 원본 외 4종 변형 데이터로 회귀 검증한다.

| 변형 | 검증 대상 |
|------|-----------|
| A | 짧은 기간·소채널 (4주·3채널) — 데이터가 적어도 동작하나 |
| B | 오가닉 매출 없음 — 오가닉 섹션 조건부 처리 |
| C | 주차 재라벨(W5–W12) — 주차 라벨 하드코딩 없나 |
| D | 12주 초과 — 롤링 8주 트림이 올바른가 |

```bash
python3 tests/verify_reproducibility.py   # 원본 + 변형 4종 전체 통과 확인
```

| 채널 평가를 내부 순위 → 시장 벤치마크 대비로 전환 | 절대 ROI 1위여도 시장 대비로는 개선여지일 수 있어 증액 오판 위험 |
| 손실 이슈와 데이터 정정 사안 분리 | 조치처가 다름(예산 재배분 vs 트래킹팀 원본 재확인) |
| 이슈 분류를 5그룹(2축) → 3그룹(레버 1축)으로 재조정 (Step 43) | "읽는 사람 입장에서 중구난방"이란 피드백 → '운영 관찰'과 '데이터 품질'이 사실상 같은 레버를 두 번 분리한 것임을 확인, 조치처 하나의 축으로 수렴 |

**차별점으로 가져간 것**
1. 과제를 그대로 풀지 않고, **데이터가 유도하는 3가지 오판을 먼저 증명한 뒤 그것을 막는 것**을 성공 기준으로 재정의했다.
2. **계산 정본 하나**로 4개 산출물(경영진/마케팅팀 × md/html)을 렌더해 수치 불일치를 설계로 차단했다.
3. 설계를 한 번에 확정하지 않고 **피드백을 근거로 3번 리팩터링**했으며(그 판단 과정을 decisions.md에 전부 기록), 특히 "이 분류가 표준인가?"라는 질문에 방어적으로 답하지 않고 커스텀 기준의 구조적 중복을 인정한 뒤 재설계로 이어갔다.
4. **판정(강건: LOO 중앙값)과 감시(민감: WoW)를 의도적으로 다른 통계 축으로 분리**해, 하나의 지표로 두 목적을 억지로 겸하지 않았다.
